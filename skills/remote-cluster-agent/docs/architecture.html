Execution Modes
Agent mode is primary. Sentinel is the automatic fallback.

Agent Mode primary

~0.1s

Persistent SSH connection to cluster-side agent.py. Commands sent as JSON, results returned immediately.

→ {"cmd": "nvidia-smi", "id": "a1b2"} ← {"output": "...", "exit_code": 0} → {"cmd": "python train.py", ...} ← {"output": "...", "exit_code": 0}

Sentinel Mode fallback

~1.5s

Per-command SSH with sentinel pattern detection. Used automatically when agent is unavailable.

→ ssh host 'cmd; echo ___EXIT_$?___' ...command output... ___MCP_EXIT_0___ ← detected → proc.kill() ← force close
