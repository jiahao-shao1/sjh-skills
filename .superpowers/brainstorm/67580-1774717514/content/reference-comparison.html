两个参考系统对比

Vin 的 Obsidian + Claude Code vs My-Brain-Is-Full-Crew，提炼出适合你的设计

🏗️ Vault 结构对比

Vin 的方式：自由链接型
vault/
  daily/ ← 每日笔记（素材流）
  contexts/ ← 深度上下文文件
    personal-workflow.md
    new-context.md (媒体公司)
    podcast-context.md
  people/ ← 人物专属笔记
    greg-eisenberg.md
  concepts/ ← 理论/模式笔记
关键特征：不用 PARA，靠 wikilinks 自然生长
链接策略：链到具体人/项目/模式，不链宽泛标签
AI 角色：只读 + 终端输出，绝不写入 Vault
My-Brain-Is-Full-Crew：PARA 自动化型
vault/
  00-Inbox/ ← 随手扔入
  01-Projects/ ← AI 自动归档
  02-Areas/ ← AI 自动归档
  03-Resources/ ← AI 自动归档
  04-Archive/ ← 只归档不删除
  07-Daily/ ← 每日笔记
  .claude/agents/ (8 个专职 Agent)
  .claude/skills/ (复杂多步技能)
关键特征：PARA + Zettelkasten 混合
归档策略：Sorter Agent 自动分类到正确文件夹
AI 角色：可读可写，但不删除，重大操作需确认

⚡ Slash Commands 对比

类别 Vin 的命令 MBIF-Crew 的对应
上下文加载 /context — 读取全局+项目上下文 + 追踪链接 Seeker Agent 自动检索相关笔记
每日规划 /today — 日历+iMessage+周笔记→当日计划 Postman 整合 Gmail/日历截止日
自我审视 /challenge 矛盾检测 · /drift 知行对比 · /emerge 潜意识提取 无直接对应（偏效率工具非反思工具）
想法演化 /trace 追踪 · /graduate 零散→独立文章 Connector 发现隐藏关联
跨域发现 /deep 30天扫描 · /connect 强制关联 Connector + Librarian 健康维护
语音克隆 /ghost — 用你的语气回答问题 无

🎯 对你的核心启示

从 Vin 借鉴

  • AI 只读 Vault — 完美匹配你的"保持自己 taste"
  • 深度上下文文件 — 为每个项目维护 context.md
  • 反思类命令 — /challenge /drift /emerge 很有思考价值
  • 链接到具体概念 而非宽泛分类

从 MBIF-Crew 借鉴

  • PARA 文件夹结构 — 你已经熟悉这套分类法
  • Inbox → 自动分类 — 低摩擦输入的关键
  • 不删除只归档 — 防御性设计
  • Agent 链式调度 — 输出标识触发下一个 Agent
